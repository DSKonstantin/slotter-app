# Правила уведомлений (Напоминания)

Этот документ описывает логику уведомлений и напоминаний как для клиентов, так и для мастеров.

## Уведомления для клиентов

Клиент получает уведомление в следующих случаях:

1.  **Запись создана**: Когда клиент оформляет новую запись.
2.  **Запись подтверждена**: Когда мастер подтверждает запись.
3.  **Запись отменена**: Когда мастер отменяет запись.
4.  **Напоминание о приеме**: Отправляется за 2 часа до начала приема.

## Уведомления для мастеров

Мастер получает уведомление в следующих случаях:

1.  **Новая запись**:
    *   **Автоподтверждение включено**: «Запись создана и/или подтверждена».
    *   **Автоподтверждение выключено**: «Создана новая запись, требуется подтверждение».
2.  **Запись отменена**: Когда клиент отменяет свою запись.
3.  **Напоминание о приходе клиента**: Отправляется за 2 часа до запланированного времени прихода клиента.

## Настройки уведомлений для мастера (Модель User)

Мастер может управлять получением уведомлений через следующие поля в модели `User`:

1.  **Новая запись** (`is_notify_new_booking`): Включает/выключает уведомления о создании новых записей.
2.  **Отмена клиентом** (`is_notify_cancellation`): Включает/выключает уведомления, когда клиент отменяет запись.
3.  **Напоминания** (`is_notify_reminder`): Включает/выключает напоминания о приходе клиента за 2 часа.
